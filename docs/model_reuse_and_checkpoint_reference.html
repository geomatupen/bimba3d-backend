Model Reuse and Checkpoint Reference

Purpose: quick reference for what appears in Model Library, what reuse does, and what files are updated.

Core Definitions

  • Checkpoint: a binary training snapshot file with learned tensors, typically .pt for gsplat (and .pth may appear in litegs paths).
  • Model entry: a registry item with id, name, source, and metadata that points to one checkpoint path.
  • Project model state: internal lineage metadata used to find the latest project checkpoint for reuse when no global model is selected.

What You See in Model Library

  • The tab shows model entries, not all raw checkpoint files.
  • One model entry can correspond to many checkpoints across runs in its lineage.
  • Baseline sessions are hidden in Session Models view because they are reference sessions, not reusable model entries.

Reuse Behavior

  • If you select a global elevated model, that specific model id is used.
  • If you do not select a global model and start mode is reuse, backend resolves the active project model lineage and picks the latest usable checkpoint.
  • Training then loads that checkpoint and continues optimization.
load previous checkpoint - train - save new checkpoint(s) in the new run

Does Reuse Update the Same Elevated Model File?

  • No, not automatically.
  • New runs save new checkpoints under that run folder.
  • The selected elevated model remains the same snapshot unless you elevate again from a newer run.

Which File Types Store What

  • .pt or .pth: model weights and tensor state used for training reuse.
  • .json: metadata, lineage, run config, and model index.

Best practice used here: binary checkpoints for weights, JSON for metadata and traceability.

Quality Metrics in Training Reports

  • PSNR (Peak Signal-to-Noise Ratio): pixel-level fidelity metric derived from reconstruction error. Higher is better.
  • SSIM (Structural Similarity): compares structure, contrast, and luminance similarity. Usually in [0, 1], and higher is better.
  • LPIPS (Learned Perceptual Image Patch Similarity): deep-feature perceptual distance. Lower is better.

Use all three together: PSNR captures numeric closeness, SSIM captures structural consistency, and LPIPS captures perceptual similarity.
