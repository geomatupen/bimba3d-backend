AI Optimization Input Modes - Implementation Reference

Reference documentation for the implemented EXIF-based AI optimization system. Features three input modes (EXIF-only, flight plan, external features) and two selector strategies (preset-bias and contextual-continuous) for automatic training parameter selection.

1 - System Overview

The AI optimization system automatically adjusts training parameters based on scene characteristics. It has two independent dimensions: input modes (what features to use) and selector strategies (how to choose parameters).

3 - Selector: Preset-Bias

File: learner.py
Strategy: Discrete preset selection with learned bias per mode.
Works with: All three input modes (but doesn't use context features directly).

How It Works

  1. Defines 4 hardcoded presets with fixed multiplier tables
  2. Maintains per-mode bias values for each preset
  3. Selects preset = argmax(bias[preset] + heuristic_bonus)
  4. After run, updates bias based on reward relative to baseline

The 4 Presets

PresetDescription
conservativeLower LR, safer for uncertain scene quality/blur/vegetation
balancedDefault middle ground (baseline multipliers = 1.0)
geometry_fastHigher geometry LR for clean structured scenes
appearance_fastHigher appearance LR when color fitting is bottleneck

Bias Update Rule

After each run with reward signal:

  • Selected preset: bias[selected] += 0.10 * (reward - reward_mean)
  • Other presets: bias[others] -= 0.015 * (reward - reward_mean)
  • All bias values clamped to [-3.0, 3.0]
Implementation: See select_preset() (line 69) and update_from_run() (line 120) in learner.py. Preset multipliers defined in common.py apply_preset_updates().
4 - Selector: Contextual-Continuous

File: contextual_continuous_learner.py
Strategy: Predicts 8 continuous multipliers using linear contextual bandit (Thompson sampling).
Works with: All three input modes (uses context vector for predictions).

How It Works

  1. Builds context vector x from input mode features (9, 19, or 29 dimensions)
  2. For each of 8 multipliers, maintains ridge regression model (A, b matrices)
  3. Predicts multiplier using θ = A⁻¹b, with Thompson sampling for exploration
  4. After run, updates all 8 models: A += x⊗x, b += reward·x

The 8 Multipliers (Continuous Range)

Multiplier Range Controls
feature_lr_mult[0.5, 1.5]SH/appearance learning speed
position_lr_init_mult[0.5, 1.5]Gaussian position updates
scaling_lr_mult[0.5, 1.5]Gaussian scale updates
opacity_lr_mult[0.5, 1.5]Opacity adaptation
rotation_lr_mult[0.5, 1.5]Rotation/quaternion updates
densify_grad_threshold_mult[0.7, 1.3]Densification trigger sensitivity
opacity_threshold_mult[0.7, 1.3]Pruning aggressiveness
lambda_dssim_mult[0.7, 1.3]DSSIM vs photometric loss weight

Model Update (Ridge Regression)

Each multiplier has independent model parameters updated after each run:

A ← A + x ⊗ x,    b ← b + reward · x(Ridge update)
θ = A-1b,    multiplier = clamp(xTθ, range)(Prediction)

Exploration vs Exploitation

Mode Behavior
thompsonSample θ ~ N(θ_mean, σ²A⁻¹) where σ² = 1/(n+1)
greedyUse posterior mean θ_mean = A⁻¹b directly
Key advantage: Learns continuous adjustments tailored to context. With more data, can discover parameter combinations beyond the 4 discrete presets.
5 - Reward Formula (Common to Both Selectors)

Both selector strategies (preset-bias and contextual-continuous) use the same reward calculation. The reward compares the current run's composite score against the baseline gsplat run.

Step 1: Find Best Quality Step

For each eval step, compute quality score:

Q(t) = 0.4·PSNR + 0.3·SSIM + 0.3·(1-LPIPS)(Quality)

Best step t_best = step with highest Q(t).

Step 2: Normalize Metrics (Joint Scaling)

For each metric, normalize using run + baseline joint range:

Norm(v) = (v - min(all)) / (max(all) - min(all))(Joint normalization)

This prevents independent scaling artifacts. LPIPS and loss are inverted (1 - Norm) so higher is better.

Step 3: Compute Composite Score

At each eval step t:

S(t) = 0.75·Q_norm(t) + 0.25·T_norm(t)(Composite)

Where T_norm(t) = 1 - clamp(elapsed(t) / baseline_time_ref, 0, 1)

Step 4: Run Decision Score

S_run = max(S(t_best), S(t_end))(Best of two anchors)
S_baseline = max(S_base(t_best_base), S_base(t_end))(Same for baseline)

Step 5: Reward Signal

reward = S_run - S_baseline(Relative improvement)

Positive reward = better than baseline. Negative = worse than baseline.

6 - Training Pipeline

File: training_pipeline_orchestrator.py
Cross-project training pipelines enable shared model learning across multiple projects.

How It Works

  1. Create pipeline with multiple projects and training configuration
  2. Set shared_model_dir in each project's config.json
  3. Pipeline orchestrator runs projects in phases with optional pass repetitions
  4. Each project trains with shared model state (updated after each run)
  5. File locking (filelock) prevents concurrent write conflicts

Shared Model Storage

SelectorShared Path
preset_biasshared_model_dir/input_mode_selector/selector_model.json
contextual_continuousshared_model_dir/contextual_continuous_selector/{mode}.json

Pipeline Execution Flow

  1. Phase 1 - Exploration: Forced preset rotation if warmup enabled
  2. Phase 2 - Stability: Adaptive selection with medium jitter
  3. Phase 3 - Convergence: Adaptive selection with minimal jitter
Key feature: Cross-project learning allows the model to generalize across different scenes. Models trained on diverse projects (camera profiles, flight geometries, scene content) can better predict parameters for new unseen projects.
